ESLINT

É para evitar problemas, pois define um padrão para o código. Serve para padroniza e para manter a consistência entre diferentes códigos js

Como instalar o Eslint:

  • npm install eslint -g -> instalar o eslint globalmente
  • npm init -> Inicia o repositório npm no local
  • eslint --init -> use popular > airbnb > (n) Enter > Json > (y) Enter ==> Para inicialização do Eslint no repositório

WEBPACK

É um bundler que agrupa / processa diversos arquivos e otimiza os mesmos. Porem é bastante configurável

Como usar usar o Webpack

  • npm install --save-dev webpack webpack-cli -> ter o package.json antes
  • npx -> é a mesma coisa que utilizar node_modules/.bin/webpack, ele facilita para chamarmos a cli´s
  • npx webpack .js/script.js -o main.js -> ele vai pegar o arquivo scripts.js e transformar em um arquivo muito menor chamado de main.js, se colocar o --watch no final, ele vai automaticamente compilar na hora para o arquivo main.js
  • No package Json, mude o test para o nome que quiser e coloque em seguida o comando de execução com watch no final, para não precisar de chamar toda hora. EX: Pois então basta fazer npm run (Nome que escolheu) e pronto ele vai copilando automaticamente para o arquivo main.js quando for alterado o arquivo script.js neste caso!

Babel

é um compilador que transforma código novo em código antigo. Por conta dos browser antigos e da para sicrnonizar com eslint e com webpack para não ficar dando erro e compilar para o arquivo main.js com código antigo para diferentes browsers suportarem!

Como usar o babel:

  • As tecnologias usadas no site, de uma olhada no site "i can use" para ver algumas das suas tecnologias usadas no código
  • Polyfill VS transpiler:
  • Polyfill => Cria métodos / funções com o nome das atuais, porem utilizando código antigo.
  • Transpiler => Transforma o código novo em antigo, tipo o const em var!

Observações: Arquivos no git ignore ==> node_modules (pois é muita memória ocupada por este arquivo)
